数据结构总统筹学习

本文档系统梳理 C 语言视角下的核心数据结构，从动态内存分配到复杂树结构，构建完整知识体系。

01 / Dynamic Memory Allocation

在学习复杂的数据结构之前，需要改变创建固定大小数组的思维定式，引入动态内存分配。

Core Concept：数组分配内存在栈（stack）中，动态分配的内存位于堆（Heap）中，需要手动调用 free(p) 释放，释放后建议将指针置为 p = NULL 以避免野指针。

三个核心函数

函数 用途 示例 特性
malloc 分配内存 int* p = (int*)malloc(10 * sizeof(int)) 内容随机，不初始化
calloc 分配并清零 int* p = (int*)calloc(10, sizeof(int)) 自动初始化为 0
realloc 调整内存大小 p = (int*)realloc(p, 20 * sizeof(int)) 可扩容 / 缩容

02 / Sequential List

数组的增强版，保留随机访问优势，同时提供动态扩容能力。

结构体定义

基本操作

操作 函数名 时间复杂度 说明
初始化 Init O(1) 分配内存，设置初始值
插入 Insert O(n) 在指定位置插入元素
删除 Delete O(n) 删除指定位置的元素
按位查找 GetByPos O(1) 通过下标直接访问
按值查找 FindByVal O(n) 遍历查找第一个匹配项
修改 Update O(1) 修改指定位置的值
销毁 Destroy O(1) 释放内存

适用场景分析

场景 原因 实例
频繁按位置访问 O(1) 随机访问 成绩单、排行榜
数据量相对固定 一次分配，无需扩容 月份天数、星期
尾部增删为主 尾部操作不需移动 日志记录、消息队列
存储密集型 空间利用率高（无指针） 图像像素、音频采样
排序后数据 二分查找需要 O(1) 访问 电话本、字典

03 / Linked List

指针串起的动态结构，牺牲随机访问速度，换取 O(1) 的插入删除。

结构体定义

基本操作

操作 函数名 时间复杂度 说明
初始化 InitList O(1) 创建头结点
头插法 InsertHead O(1) 在头部插入
尾插法 InsertTail O(1)* 需尾指针
指定位置插入 Insert O(n) 遍历到 pos-1
头删 DeleteHead O(1) 删除第一个结点
尾删 DeleteTail O(n) 遍历到倒数第二个
指定位置删除 Delete O(n) 遍历到 pos-1
按位查找 Get O(n) 遍历到第 i 个
按值查找 Find O(n) 遍历直到找到
销毁 DestroyList O(n) 逐个释放结点

注：尾插 O(1) 的前提是链表持有 tail 指针。

链表的变体

类型 结构特点 适用场景
单向链表 只有 next 指针 大多数场景
双向链表 prev + next 反向遍历、O(1) 删除指定结点
循环链表 尾结点指向头结点 约瑟夫环、轮询调度
静态链表 数组模拟 不支持指针的低级环境

适用场景分析

场景 原因 实例
频繁插入/删除 O(1) 时间，不移动元素 任务调度、消息队列
数据量不确定 按需分配，无容量概念 动态数据采集
内存碎片化 不需要连续大块内存 嵌入式系统
多项式运算 稀疏存储非零项 科学计算
LRU 缓存 移动结点到头部 O(1) Redis、浏览器缓存

顺序栈结构体定义

查看栈顶 GetTop / Peek O(1) 只读不删

撤销操作 后做操作先撤销 Ctrl+Z

05 / Queue

先进先出（FIFO）。

循环队列结构体定义

Note：循环队列扩容不能直接使用 realloc，因元素在数组中不连续。需 malloc 新内存后手动复制。

扩容实现

核心操作

操作 函数名 时间复杂度 循环队列特有条件
初始化 InitQueue O(1) front = rear = 0
入队 Enqueue O(1) 均摊 rear = (rear+1) % capacity
出队 Dequeue O(1) front = (front+1) % capacity
判空 IsEmpty O(1) front == rear
判满 IsFull O(1) (rear+1) % capacity == front

适用场景分析

场景 原因 实例
固定大小缓冲区 环形复用，不扩容 串口缓冲、音频缓冲
生产者-消费者模型 一写一读，环形同步 多线程任务队列
滑动窗口 窗口固定大小 网络流量统计
最近 N 条记录 只保留最新 游戏 Replay、日志缓存
嵌入式系统 内存受限 单片机数据采集

08 / Red-Black Tree

近似平衡的二叉搜索树，通过颜色标记与旋转维持平衡，牺牲少量查找性能换取更快的插入删除。

五条性质

编号 性质
1 结点非红即黑
2 根为黑色
3 叶结点（NIL）为黑色
4 红结点的子结点均为黑（无连续红）
5 任一结点到叶子的所有路径黑结点数相同

红黑树 vs AVL

对比项 红黑树 AVL
平衡条件 黑高平衡 高度差 ≤ 1
树高 ≈ 2 log n ≈ 1.44 log n
查找 略慢 更快
插入/删除 更快（旋转少） 较慢
适用场景 读写均衡 读多写少

结构体定义

应用场景

应用 说明
C++ STL map / set 底层实现
Java TreeMap / TreeSet -
Linux 调度器 进程管理
epoll 文件描述符管理
Nginx 定时器管理

数据结构 访问 查找 插入 删除

顺序表 O(1) O(n) O(n) O(n)
链表 O(n) O(n) O(1)* O(1)*
栈 O(n) O(n) O(1) O(1)
队列 O(n) O(n) O(1) O(1)
哈希表 N/A O(1) O(1) O(1)
BST O(log n) O(log n) O(log n) O(log n)
AVL O(log n) O(log n) O(log n) O(log n)
红黑树 O(log n) O(log n) O(log n) O(log n)

*已知位置的前提下

10 / Learning Path

  1. Foundation：顺序表 + 链表，理解堆与栈的内存模型
  2. Abstraction：掌握每种结构的核心哲学（LIFO / FIFO / 平衡）
  3. Implementation：手写核心操作，不依赖现成库
  4. Scenario-Driven：遇到问题时思考“哪种结构最匹配”
  5. Complexity Analysis：养成分析时间 / 空间开销的习惯
